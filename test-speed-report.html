What Changed

Fix 1 — Scoped per-lib tsconfig commit a2fe5a0

PR #5607 deleted per-lib tsconfig.spec.json files; every lib now pointed at libs/dh/tsconfig.spec.json which has no include restriction. readConfiguration returned all ~750 .ts files as NgtscProgram rootNames per lib (3000ms init). Fix: libs/dh/vite.config.mts now writes a per-lib tsconfig.vitest.json scoping files/include to ~5 rootNames. Also sets moduleResolution: node to drop the extra 600ms that Bundler adds. Transform time: 5.2s → 2.1s.

Fix 2 — Removed server.deps.inline: [/fesm2022/] commit a2fe5a0

Adding fesm2022 inline forced Vite to re-transform all 231 Angular fesm2022 packages on every lib run (~2.8s overhead). The original config had no inline rule (55ms). Removed from all three configs; isolate: false continues to share the module graph without it.

Fix 3 — --parallel=4 in test scripts commit f5f9bbc

nx.json global "parallel": 1 serialised all 35 lib test processes. Each lib uses maxWorkers: 1 internally, so 4 can safely run concurrently on GitHub's 4-core runner. Added --parallel=4 to affected:test and test scripts in package.json. CI wall-clock: 7m 3s → 1m 17s.
